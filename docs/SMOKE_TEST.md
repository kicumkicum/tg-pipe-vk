# Smoke test

## Подготовка

- Убедись, что Vercel env vars заполнены (см. `docs/DEPLOY.md`).
- Убедись, что:
  - VK Callback API настроен на `/api/vk` и подтверждён
  - Telegram webhook настроен на `/api/telegram`

## Проверки

1) **VK → TG**

- Отправь обычный текст в VK беседе.
- Убедись, что в Telegram пришло сообщение с заголовком (имя + ссылка «VK») и текстом ниже (HTML-форматирование).

2) **TG → VK**

- Отправь обычный текст в Telegram группе.
- Убедись, что в VK пришло сообщение с префиксом `[TG | …]` и тем же текстом.

3) **Анти-цикл**

- Сообщения от **бота** в Telegram не должны уходить обратно в VK (апдейты с `from.is_bot`).
- Исходящие сообщения в VK с флагом **`out`** (наши отправки TG→VK) не должны снова попадать в Telegram.
- Старые сообщения с текстом `[BRIDGE:` по-прежнему игнорируются (легаси).

4) **Повтор webhook**

- Повтори отправку похожих сообщений и проверь, что нет «пинг‑понга» между чатами.
